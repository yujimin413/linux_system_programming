# Linux System Programming Utilities

리눅스 환경에서 **파일 시스템, 프로세스, 시그널, 디렉토리 순회, 파일 비교** 등
핵심 시스템 프로그래밍 요소를 구현한 **유틸리티 프로그램 모음**입니다.

각 디렉토리는 독립적인 실행 프로그램으로 구성되어 있으며,
리눅스 시스템 자원을 제어하고 관리하는 방식에 초점을 맞춰 설계되었습니다.

## 1️⃣ backup — 파일·디렉토리 백업 / 복구 시스템

파일 또는 디렉토리를 백업 디렉토리로 복사·관리하고,
중복 여부를 **해시(md5 / sha1)** 기반으로 판단하여 백업 무결성을 보장하는 백업 유틸리티입니다.

### 핵심 특징

* 파일 / 디렉토리 백업 지원 (`-d` 옵션 시 재귀 백업)
* `md5` 또는 `sha1` 해시를 이용한 **중복 백업 방지**
* 동일 파일 다중 백업 시 **시간 스탬프 기반 관리**
* 백업 파일 선택 삭제 / 전체 삭제 / 복구 지원
* 모든 명령은 **내장 커맨드 기반 CLI**로 동작

### 디렉토리 구조 요약

```text
backup/
├── ssu_backup      # 메인 실행 파일
├── start.c/.h     # 프롬프트 및 명령 처리
├── add.c          # 백업(add) 로직
├── recover.c      # 복구(recover) 로직
├── md5.c / sha1.c # 해시 계산
├── help.c         # 도움말 출력
└── Makefile
```

### 주요 시스템 개념

* 파일 I/O, 디렉토리 순회
* 해시 기반 파일 비교
* 명령어 파싱 및 예외 처리
* 단일 프로세스 흐름 제어

## 2️⃣ monitoring — 디렉토리 변경 감지 모니터링

지정한 디렉토리를 **디몬(daemon) 프로세스**로 상시 감시하며,
파일의 생성·수정·삭제 이벤트를 감지하여 로그로 기록하는 모니터링 프로그램입니다.

### 핵심 특징

* 디렉토리마다 **독립적인 디몬 프로세스 생성**
* 주기적 스캔을 통한 상태 트리(old/new) 비교
* 생성(create) / 수정(modify) / 삭제(remove) 이벤트 감지
* 시그널(`SIGUSR1`) 기반 안전 종료
* 모니터링 목록을 `monitor_list.txt`로 관리

### 디렉토리 구조 요약

```text
monitoring/
├── ssu_monitor     # 실행 파일
├── ssu_monitor.c   # 메인 로직
├── ssu_monitor.h   # 자료구조 / 선언
├── monitor_list.txt# 모니터링 디렉토리 목록
└── jimindir/       # 모니터링 대상 예시 디렉토리
```

### 주요 시스템 개념

* 디몬 프로세스 생성
* 시그널 처리
* 디렉토리 트리 비교
* 로그 파일 관리

## 3️⃣ scoring — 시험 답안 자동 채점 시스템

학생 제출 디렉토리(STD_DIR)와 정답 디렉토리(ANS_DIR)를 기준으로
**빈칸 채우기 문제 + 프로그램 작성 문제**를 자동으로 채점하는 시스템입니다.

### 핵심 특징

* 최대 100문제까지 채점 가능
* 빈칸 문제: 문자열 비교 + 다중 정답 처리
* 프로그램 문제:

  * 컴파일 → 실행 → 실행 결과(stdout) 비교
  * 공백·대소문자 무시 비교
  * 실행 시간 초과 / 컴파일 에러 감점 처리
* 점수 테이블(`score_table.csv`) 자동 생성 및 수정
* 채점 결과를 `score.csv`로 저장

### 디렉토리 구조 요약

```text
scoring/
├── ssu_score.c/.h # 채점 메인 로직
├── main.c         # 프로그램 진입점
├── blank.c/.h     # 빈칸 문제 처리
├── ANS_DIR/       # 정답 파일 디렉토리
├── STD_DIR/       # 학생 제출 디렉토리
└── Makefile
```

### 주요 시스템 개념

* 파일 비교 및 파싱
* 프로세스 실행 결과 관리
* 링크드 리스트 기반 데이터 관리
* CSV 파일 생성 및 처리
